> **Origin of the Name**: *Alphalith* = **Alpha** (excess returns) + **Lith** (Ancient Greek λίθος, "bedrock") → "Alpha decisions sealed in the bedrock," echoed by the closing inscription *Sealed in the Bedrock*. The Chinese name 慧投 (Huìtóu) = 慧眼 (insightful eye) + 投研 (investment research).

> A lightweight, zero-dependency multi-agent AI investment engine with native support for **A-share / HK / US** markets.

> Hedge funds have analyst teams. **Now you do too.**

**Alphalith** convenes a council of AI analysts—technical, fundamental, news, and sentiment—to debate every trade and produce reproducible, accountable decisions. One command. Three markets. Zero excuses.

## ✨ Why Alphalith

| Problem | Alphalith's Answer |
|---------|--------------------|
| Existing AI trading frameworks need 13 API keys + Conda + 30 min setup | `pip install alphalith` and you're running in **30 seconds** |
| Single-LLM analysis is shallow and hallucination-prone | **4 specialized agents** debate before deciding |
| Cost is too high for retail ($0.5+/run) | DeepSeek default → **<$0.01 per analysis** |
| A-share / HK quirks (T+1, lot size, stamp duty) ignored | Full **market-rule engine** built-in |
| Foreign tools, English-only, hard for Chinese users | **Bilingual** prompts & UI from day one |
| No path to live trading | **TradingView Webhook** native support |

Start the GUI — no extra dependencies needed:

alphalith gui --port 28080   # custom port

## 🌍 Three Markets, One Workflow

| Feature | 🇨🇳 A-share | 🇭🇰 Hong Kong | 🇺🇸 US Stocks |
|---|---|---|---|
| Code formats | `300308`, `中际旭创`, `300308.SZ` | `0700`, `腾讯`, `0700.HK` | `NVDA`, `英伟达`, `NASDAQ:NVDA` |
| Settlement | T+1 | T+0 | T+0 |
| Price limits | ±10% / ±20% / ±5% (ST) | None | None (circuit breakers only) |

## 🛠 Architecture

```
┌────────────────────────────────────────────────────────────┐
│  CLI · Web GUI · Python API · Telegram/Lark Bot            │
├────────────────────────────────────────────────────────────┤
│         Alphalith Core (7-layer 13-node pipeline)          │
│   4 Analysts → Summariser → Debate → Manager → Trader →   │
│              Triple Risk Review → Fund Manager              │
├────────────────────────────────────────────────────────────┤
│  Data: Sina · Tencent · Eastmoney · AkShare · yfinance     │
│        + 雪球 · Reddit (sentiment)                          │
│  LLM:  DeepSeek → Volcano → Stub (auto-fallback)           │
│  Out:  JSON · Markdown · HTML · Webhook                    │
└────────────────────────────────────────────────────────────┘
```

## 📦 Project Structure

```
alphalith/
├── alphalith/
│   ├── core.py          # analyze() entry point
│   ├── market.py        # 3-market detection + CN name resolver
│   ├── data.py          # Quote / news / fundamentals provider
│   ├── rules.py         # A/HK/US market rule engine
│   ├── agents.py        # 7-layer 13-node analysts + debate + risk + fund manager
│   ├── llm.py           # Fallback chain + token counting
│   ├── schema.py        # ADP v1.0 Decision dataclass
│   ├── backtest.py      # Backtest engine (7 strategies + benchmark + risk metrics)
│   ├── html_report.py   # HTML visualization report
│   ├── journal.py       # SQLite decision journal
│   ├── report.py        # Text report renderer
│   ├── cli.py           # CLI entry point
│   ├── dashboard.py     # Dashboard (quotes + signals + heatmap)
│   ├── watchlist.py     # Multi-list watchlist management
│   ├── portfolio.py     # Position tracking + P&L calculation
│   ├── sentiment.py     # Multi-source sentiment analysis (6 built-in + custom)
│   ├── alerts.py        # Price/change threshold alerts
│   ├── gui/
│   │   ├── __init__.py  # GUI HTTP server
│   │   └── app.html     # GUI frontend (single file, inline CSS/JS)
│   ├── docs/
│   │   └── CLI.md       # CLI manual
│   └── tests/
├── pyproject.toml
└── README.md
```
